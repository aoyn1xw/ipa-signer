Architecture

Main components

  • app.js: Express API, file handling, cleanup scheduler, install page generation.
  • zsign-worker.js: Worker thread wrapper for zsign signing job.
  • uploads/: runtime storage for temporary files, certs, profiles, signed IPAs, and plist files.
  • logs/: error and combined logs via Winston.

Request lifecycle

  1. Receive multipart upload with IPA + signing assets.
  2. Move files into working directories.
  3. Optionally mutate IPA with cyan.
  4. Sign IPA with zsign in a worker thread.
  5. Extract app metadata from resulting IPA.
  6. Create plist + install URL and return response.
  7. Clean transient files on timer and expiry jobs.

Operational notes

  • Rate limiting is global and controlled by env vars.
  • Generated install links and artifacts are intentionally short-lived.
  • Set PUBLIC_DOMAIN correctly in production; install URLs depend on it.
  • Cleanup timer removes stale files in temp/cert/profile folders every 5 minutes.
  • Signing executes in a worker thread to avoid blocking the main request loop.
