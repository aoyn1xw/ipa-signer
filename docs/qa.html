Q&A

Q: Why do I get “P12 and MobileProvision required”?

A: /sign requires all three files: IPA, P12, and MobileProvision. Verify multipart field names are exactly ipa, p12, and mobileprovision.

Q: Why does install link open but app will not install?

A: Usually PUBLIC_DOMAIN is wrong or not publicly reachable over HTTPS. The device must download both plist and IPA from that domain.

Q: What does “cyan not found” mean?

A: The server tried advanced tweaks but could not execute cyan. Install cyan or set CYAN_CMD to the correct binary path.

Q: Can I disable advanced tweaks and only sign?

A: Yes. Don’t send any adv_* fields. The server will sign directly with zsign.

Q: How long do generated install links stay valid?

A: One hour. Metadata and artifacts are cleaned up automatically after expiry.

Q: How do I make uploads safer?

A: Lower max request rate, run behind authentication/reverse proxy, and isolate storage volumes for uploaded secrets.
